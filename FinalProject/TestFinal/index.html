Riot Games How-To

A short guide
to using Riot's API

About this guide

This guide is meant to be a basic introduction to API provide by Riot Games for League of Legends. As such this guide will not cover everything nor will it cover the basic set up of your programming environment.
This guide will primarily show you how to:

  • Retrieve your api key from Riot
  • Make calls to retrieve a player's (summoner's) information based on their name
  • Make calls to retrieve a summoner's match history based on the information retrieved from the last request
  • Give you an idea of what to do with all of this information

There is a lot of data we could retrieve from the Riot servers which can be manipulated in many different ways. Focusing on these basics will provide you with a good platform and general knowledge to create more complex websites using this API.

Summoner Info

Primarily we access Riot's API through get requests sent to their servers. What this request looks like changes based on what information we are trying to get but usually takes on a specific form each time:

https://{region}.api.pvp.net/api/lol/{region}/{request specific pathing}/{search term}?api_key={your api key}

The first bracketed component is pretty straight forward. League of Legends is a multi-region game so you must specify which one you'd like. Next up is the {request specific pathing} which denotes what type of data we are looking form. This is the part that can look the most diverse depending on the data we seek. For the purpose of retrieving a summoner's information we will be looking at this one:

/api/lol/na/v1.4/summoner/by-name/{summonerNames}

It's not too scary to look at and should be pretty straight forward. Basically it takes in a string of the summoner's name sends back a JSON object chalk full of information. So let's look at an example of how to retrieve one of these objects from riot.

So to start I'm going to use my own summoner name "Eboncross" to demonstrate how we'd retrieve the summoner data. Now the way that I ended up getting user input was by use of a form which submits post data back to the page with my inputed summoner name. I'm going to skip all of
that though as it's not really in the scope of this how-to. When my handler detects that a summoner's name has been entered it sends a request to the Riot servers that looks like this:

request('https://na.api.pvp.net/api/lol/na/v1.4/summoner/by-name/'+ req.body.name +'?api_key={my api key}', function(err, response, body){...}

This pretty much mirrors what we talked about above and is pretty straight forward. "req.body.name" is information from my form aka the summoner name and "{my api key}" is my literal API key without the brackets. When we send this off to Riot we get an object back that looks like this:

{"eboncross": {
"id": 19089071,
"name": "Eboncross",
"profileIconId": 592,
"revisionDate": 1449036186000,
"summonerLevel": 30}}

What are we looking at here? Well what we have is an object named "eboncross" which is the search term we used. This object contains other references or foreign keys that we could then use in other queries such as my summoner id "19089071", the id for the profile icon I use, etc. Pretty easy to understand naming conventions have been used.

Next up is to show how we take this object and turn into something we can actually use with other requests.

Getting the Match History

In this section of the guide we are going to look at how to make a request for a particular summoner's match history based on the summoner information we recieved from our last request. Specifically we will be using the summoer's id. As a preview this is the request format we will be using this time:

/api/lol/na/v2.2/matchlist/by-summoner/{summonerId}

Again it is pretty straight forward and looks a lot like the prvious request. Now heading right back to where we left off last time let's look at what we need to do our previous request to make it usable in our new request:

var summoner = JSON.parse(body);
for (i in summoner){
var summonerID = summoner[i]["id"];}

Let's take a closer look at what this is doing. First we have the line var summoner = JSON.parse(body); . Essentially this takes the information sent back to us from Riot and houses it in a variable called summoner. Next part of the the code cycles through that variable to create a new house called summonerID for my summoner's id. Now what I didn't mention before is that you can actually use the request to search for multiple summoners at a time. This is why the object returned from Riot uses our search term as a name. It is possible to get multiple objects each representing a different summoner. This is also why we must us a loop to create the new variable. There is often a time you would need to search for more than one summoner at a time so the above code is designed to work for onle one. The next step in building our match history looks much like the prvious step:

request('https://na.api.pvp.net/api/lol/na/v2.2/matchlist/by-summoner/' + summonerID + '?api_key={my api key}', function(err, response, body){});

Basically all we are doing is using the previously created summonerID variable in this request. What this will return is the following:

{"matches": [ {
"timestamp": 1439450312819,
"champion": 222,
"region": "NA",
"queue": "RANKED_SOLO_5x5",
"season": "SEASON2015",
"matchId": 1917929861,
"role": "DUO_CARRY",
"platformId": "NA1",
"lane": "BOTTOM"}, ....

This will repeat over and over for each individual match in your match history giving you access to a lot of information. Next up I'll show you what you can do with all this data.

Now What?

Basically what I have shown you today is how to make a request to Riot's API for data and then how to use that data in future reqeusts. It doesn't seem like much but really it's the basic foundation for everything the Riot API allows you to do. For example you can now take the matchID information you pulled from the match history request and make another request to /api/lol/na/v2.2/match/{matchId} to get the specific match details. If you keep using these building blocks eventually you can make an awesome website like lolking.com which is etirely based on what I just showed you.
